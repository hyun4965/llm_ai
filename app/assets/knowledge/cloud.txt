클라우드(Cloud):
필요한 컴퓨팅 자원(서버, 스토리지, 네트워크, DB 등)을 인터넷을 통해
온디맨드로 빌려 쓰는 방식. 사용량 기반 과금이 일반적이다.

IaaS:
가상 서버, 네트워크 같은 인프라를 제공한다.
사용자는 OS부터 직접 관리하는 경우가 많다.

PaaS:
애플리케이션 실행 환경을 제공한다.
런타임, 배포, 스케일링 일부를 플랫폼이 관리해 운영 부담이 줄어든다.

SaaS:
완성된 소프트웨어를 서비스 형태로 제공한다.
사용자는 기능을 사용만 하고 인프라/운영은 제공자가 담당한다.

클라우드 네이티브:
컨테이너, 오토스케일, 관리형 서비스, 장애 대비 등을 전제로
클라우드 환경에 최적화된 방식으로 애플리케이션을 설계/운영하는 접근이다.

DBaaS(관리형 DB):
DB 설치/패치/백업/복구/모니터링 등을 클라우드가 대신 해주는 DB 서비스.
예: AWS RDS, Cloud SQL 같은 형태.

온프레미스(On-Premises):
회사가 직접 보유한 물리 서버나 자체 데이터센터에 시스템을 운영하는 방식.
보안/통제가 강하지만 초기 비용과 운영 부담이 크다.

VPC:
클라우드 안에서 논리적으로 분리된 사설 네트워크 공간.
서브넷, 라우팅, 방화벽 규칙 등을 구성한다.

서브넷(Subnet):
VPC 안의 더 작은 네트워크 구역.
퍼블릭 서브넷(인터넷 연결)과 프라이빗 서브넷(내부 전용)으로 나누어 운영한다.

보안그룹(Security Group):
인스턴스 단위로 적용되는 가상 방화벽.
인바운드/아웃바운드 트래픽 허용 규칙을 설정한다.

NACL:
서브넷 단위로 적용되는 네트워크 방화벽.
보안그룹보다 더 낮은 레벨에서 동작한다.

로드밸런서(Load Balancer):
트래픽을 여러 서버로 분산해 성능과 가용성을 높이는 구성 요소.
서버 한 대가 죽어도 다른 서버로 우회할 수 있게 만든다.

오토스케일링(Auto Scaling):
트래픽이 증가하면 서버를 자동으로 늘리고,
감소하면 서버를 줄여 비용과 성능을 균형 있게 유지하는 기능.

스냅샷(Snapshot):
특정 시점의 디스크/DB 상태를 저장한 백업 단위.
장애 복구나 마이그레이션에 자주 사용된다.

오브젝트 스토리지(S3 같은 것):
파일을 객체 단위로 저장하는 스토리지.
이미지/동영상/첨부파일 저장에 적합하고 확장성이 높다.

프리사인드 URL:
S3 같은 오브젝트 스토리지에 직접 업로드/다운로드할 수 있도록
짧은 시간만 유효한 접근 URL을 발급하는 방식.
서버 부하와 비용을 줄이는 데 유리하다.

멀티파트 업로드:
큰 파일을 여러 조각으로 나눠 병렬로 업로드하는 방식.
네트워크가 불안정해도 재시도에 강하다.

CDN:
사용자와 가까운 엣지 서버에서 정적 자원을 캐싱하여
응답 속도를 높이고 원본 서버 부하를 줄이는 네트워크.

DNS:
도메인 이름을 IP 주소로 바꿔주는 시스템.
온프레미스든 클라우드든 서비스 공개 시 필수로 엮인다.

모니터링/로깅:
서비스 상태(CPU, 메모리, 지연시간, 에러율)를 관찰하고
로그를 수집해 장애 원인을 추적하는 운영의 기본 요소.

배포 전략:
롤링 배포(순차 교체), 블루-그린(두 환경 전환), 카나리(일부 트래픽만 신규)
등이 있으며, 서비스 중단을 줄이기 위한 방식들이다.
